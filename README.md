# VidGram Master Documentation

## Project Overview

**What the project does:**
VidGram is a comprehensive influencer marketing platform designed to manage campaigns, assess creator authenticity, and estimate financial risk from fraudulent audience engagement. The system converts raw creator metadata into quantified fraud signals, producing actionable risk classifications for both our backend systems and the interactive frontend dashboard.

**Problem being solved:**
Influencer marketing campaigns are vulnerable to fraud through purchased followers, bot engagement, fake comments, and audience manipulation. Marketing teams need rapid, explainable risk assessments to protect campaign budgets and ensure authentic reach. This system provides a full-stack solution from data collection and fraud detection to visual campaign management.

---

## Frontend

The `frontend` directory contains the user interface for the VidGram platform. It provides a sleek, modern dashboard for campaign managers to monitor and manage their influencer marketing efforts and visualize the risk scores generated by the backend engines.

### Features
- **Dashboard (`index.html`)**: The main overview of active campaigns, total budget spent, and high-level risk metrics.
- **Analyze (`analyze.html`)**: Tools for analyzing potential influencers, displaying their authenticity scores, and calculating genuine vs. fake reach.
- **Monitor (`monitor.html`)**: Live monitoring of ongoing campaigns and influencer performance.
- **Alerts (`alerts.html`)**: Notification center for high-risk flags, budget overruns, or sudden drops in engagement based on backend signals.
- **Budget (`budget.html`)**: Financial overview to track campaign spending and visualize potential budget waste from fraudulent reach.
- **Reports (`reports.html`)**: Detailed analytics and downloadable reports on campaign success.
- **Timeline (`timeline.html`)**: A visual schedule of upcoming influencer posts and campaign milestones.
- **Settings (`settings.html`)**: Application configuration and user preferences.

### Technology Stack
- **HTML/CSS/JavaScript**: Vanilla web technologies for a lightweight, fast, and responsive user experience.

## Backend 1

Backend 1 contains the core Python-based **Fraud Detection Engine**. It analyzes influencer profiles and campaigns to assess authenticity and estimate financial risk.

### Architecture Highlights
- **Public API Layer (`api.py`)**: The single public entrypoint for Backend 2 to consume.
- **Normalization Layer**: Standardizes raw dataset formats into a consistent schema.
- **Fraud Detection Pipeline**: Five independent detectors running in parallel:
  1. **Growth Detector**: Detects 7-day follower spikes.
  2. **Engagement Detector**: Finds engagement rate anomalies.
  3. **Comment Detector**: Identifies spam and duplicate comments.
  4. **Audience Quality Detector**: Calculates fake/bot/inactive ratios.
  5. **Consistency Detector**: Monitors posting cadence stability.
- **Scoring Engine**: Aggregates the 5 detectors (Growth 20%, Engagement 25%, Comment 15%, Audience 25%, Consistency 15%).
- **ML Risk Engine**: A logistic regression model that extracts 9 features to produce a fraud probability and score adjustment.

### Integration
Backend 2 calls the `score_influencer_payload` function to process creator metadata and returns a `ScoringResult` containing the final authenticity score (0-100), risk level (Low/Medium/High), evidence chains, and campaign reach estimates.

## Backend 2

The `backend2/risk-engine` directory serves as the orchestration and financial risk layer built in TypeScript. It integrates the findings from the fraud detection models and calculates the direct financial implications for our campaigns.

### Features
- **Authenticity Classification**: Categorizes influencers into actionable risk categories (Shortlist, Monitor, Pause, Reject) based on the raw authenticity scores provided by the fraud detection pipeline.
- **Reach Calculation**: Formulas to determine how much of an influencer's audience is genuine versus fake.
- **Financial Risk Analysis**: Calculates potential budget loss and the true cost per *genuine* follower so we can make data-driven financial decisions.
- **Influencer Comparison**: Evaluates multiple influencers side-by-side to recommend the safest choice and highlight potential budget savings.

### Technology Stack
- **Node.js**: The runtime environment for our logic.
- **TypeScript**: Provides strong typing and object-oriented principles to ensure our risk calculations are robust and bug-free.
- **ts-node**: Used to execute the TypeScript files directly during development.

### How to Run the Backend 2 Risk Engine Examples
1. Navigate to the risk engine directory:
   ```bash
   cd backend2/risk-engine
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Run the usage example:
   ```bash
   npx ts-node src/examples/usage.ts
   ```
